# Repository Artifact Audit

This audit records untracked or vendored artifacts that should be reviewed
before committing the HDF5 faithfulness work.

## Keep

- `TODO.md`: tracks completed audit items and remaining compatibility work.
- `analysis/`: documents unsupported features, tracehash parity work, and this
  audit.
- `scripts/`: contains tracehash runners and C-library fixture generation.
- `tools/tracehash/`: optional local checkout of the
  [`tracehash-rs`](https://crates.io/crates/tracehash-rs) crate, gitignored.
  The Rust-side `tracehash` feature pulls the library from crates.io; this
  local checkout is only needed when building the patched libhdf5 C-side
  probes, which `#include "../../tools/tracehash/c/tracehash_c.h"`.
- `hdf5/`: vendored HDF5 C source used for source-level comparison, patched-C
  tracehash probe target selection, and fixture-generation parity checks. The
  exact source revision is recorded in `hdf5-source.json`.

## Do Not Commit Without Review

- `.codex`: local agent metadata.
- `.README.md.swp`: editor swap file. This should be removed by the owner of
  the editing session or deliberately ignored.
- Local tracehash outputs such as `rust.tsv`, `c.tsv`, and divergence reports
  regenerated during development.
- Temporary HDF5 files outside `tests/data/hdf5_ref/`.
- `target/`: build output, already ignored.
- `Cargo.lock`: ignored for the root crate in this repository.

## Generated Fixtures

The modern chunk-index fixture files in `tests/data/hdf5_ref/` are generated by
`scripts/generate-modern-fixtures.py` using h5py/HDF5 1.14.5 with
`libver="latest"`.
